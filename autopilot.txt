Use the `autopilot` tool as the session-scoped autopilot command surface.

Interpret the request like this:
- `on` => `autopilot(action="on")`
- `off` or `stop` => `autopilot(action="off")`
- `status` => `autopilot(action="status")`
- `help` => `autopilot(action="help")`
- anything else => `autopilot(task="<full request>")`

Rules:
- Prefer slash-command semantics over the old control-agent workflow.
- If the request includes `allow-all` or `allow all`, set `permissionMode="allow-all"`.
- If the request includes `limited`, set `permissionMode="limited"`.
- If the request includes a cap such as `max 3`, `max=3`, or `continue 3 times`, pass it as `maxContinues`.
- If the request includes an agent override such as `agent build-high`, `use build-high`, or `agent=build-high`, pass it as `workerAgent`.
- When autopilot is enabled without a task, the goal is to make the rest of the session more autonomous and reduce follow-up questions.
- When a task is provided, the configured delegate agent acts as the orchestrator that keeps the task moving until it completes, blocks, or hits the continuation limit.
